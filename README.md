# South of France Trip — Private Travel Plan

A single-page, password-protected travel itinerary, ready to host on **GitHub Pages**.

The published page (`index.html`) is **encrypted**: the itinerary is AES-256-GCM
encrypted with your password and only decrypted in the browser *after* the correct
password is entered. The plaintext never ships in the hosted file.

## Files

| File | Purpose |
|------|---------|
| `index.html` | The published password gate (HTML only). Generated by the build — commit this. |
| `src/gate.html` | Gate page template. Build injects the encrypted payload here. |
| `src/content.html` | The real itinerary (HTML only). **Git-ignored** — keep a backup! |
| `src/stargazing.html`, `src/shopping.html` | Extra page fragments injected at build time. |
| `supabase/shopping_items.sql` | Supabase table + RLS policies for the shared shopping list. |
| `assets/gate.css`, `assets/gate.js` | Styles and unlock logic for the password gate. |
| `assets/content.css`, `assets/content.js` | Styles and interactivity for the itinerary page. |
| `build.mjs` | Encrypts `src/content.html` → `index.html` using your password. |
| `package.json` | Convenience `build` script. |
| `.nojekyll` | Tells GitHub Pages to serve files as-is. |

## Build / set the password

Requires **Node 22** (uses the built-in Web Crypto API — no dependencies to install).

```bash
node build.mjs "your-password-here"
# or
PASSWORD="your-password-here" node build.mjs
```

This (re)generates `index.html`. To change the password later, just re-run with a new one.
To edit the plan, edit `src/content.html` (markup), `assets/content.css` (styles), or `assets/content.js` (countdown, animations), then rebuild.

## Shared shopping list

The **Shopping** page (`#shopping`) lets everyone add villa groceries and tick items off when bought. Purchased items are hidden by default; use **Show purchased** to review them.

By default the list is stored in each browser (`localStorage`) — fine for trying it out, but not shared across devices. To sync for the whole group via [Supabase](https://supabase.com/):

1. Create a Supabase project.
2. Run `supabase/shopping_items.sql` in the SQL editor (creates the table and row-level security policies).
3. In **Database → Replication**, enable **Realtime** for `shopping_items`.
4. From **Project Settings → API**, copy the **Project URL** and **anon public** key.
5. Paste them into `data-supabase-url` and `data-supabase-anon-key` on the `#page-shopping` element in `src/shopping.html`.
6. Rebuild and redeploy.

The anon key lives inside the encrypted page (only visible after unlock). RLS allows anyone with the key to read and update the list — acceptable for a password-protected trip site; tighten policies later if needed.

There is no cap on how many items can be added.

## Deploy to GitHub Pages

```bash
git init
git add .
git commit -m "Private travel plan"
git branch -M main
git remote add origin git@github.com:<you>/<repo>.git
git push -u origin main
```

Then in the repo: **Settings → Pages → Source: Deploy from a branch**, branch **main**,
folder **/ (root)**. The site appears at `https://<you>.github.io/<repo>/` within a minute or two.

## Notes & limitations

- Choose a **strong password** — anyone with the URL can attempt to unlock it, and
  client-side protection on a static host can't rate-limit guesses. The encryption is
  only as good as the password.
- `src/content.html` is git-ignored on purpose, so the plaintext plan stays off GitHub.
  **Keep your own backup of it** — it's the only editable source.
- Decryption needs a **secure context**: works on GitHub Pages (HTTPS) and `localhost`,
  but opening `index.html` straight from disk (`file://`) may fail. To test locally:
  ```bash
  npx serve .        # or:  python3 -m http.server
  ```
- No password recovery — if forgotten, just rebuild with a new password.
