# South of France Trip — Private Travel Plan

A single-page, password-protected travel itinerary, ready to host on **GitHub Pages**.

The published page (`index.html`) is **encrypted**: the itinerary is AES-256-GCM
encrypted with your password and only decrypted in the browser *after* the correct
password is entered. The plaintext never ships in the hosted file.

## Files

| File | Purpose |
|------|---------|
| `index.html` | The published, **encrypted** page. Generated by the build — commit this. |
| `src/content.html` | The real itinerary (plaintext). **Git-ignored** so it never reaches the repo. Keep a backup! |
| `build.mjs` | Encrypts `src/content.html` → `index.html` using your password. |
| `package.json` | Convenience `build` script. |
| `.nojekyll` | Tells GitHub Pages to serve files as-is. |

## Build / set the password

Requires **Node 18+** (uses the built-in Web Crypto API — no dependencies to install).

```bash
node build.mjs "your-password-here"
# or
PASSWORD="your-password-here" node build.mjs
```

This (re)generates `index.html`. To change the password later, just re-run with a new one.
To edit the plan, edit `src/content.html` and rebuild.

## Deploy to GitHub Pages

```bash
git init
git add .
git commit -m "Private travel plan"
git branch -M main
git remote add origin git@github.com:<you>/<repo>.git
git push -u origin main
```

Then in the repo: **Settings → Pages → Source: Deploy from a branch**, branch **main**,
folder **/ (root)**. The site appears at `https://<you>.github.io/<repo>/` within a minute or two.

## Notes & limitations

- Choose a **strong password** — anyone with the URL can attempt to unlock it, and
  client-side protection on a static host can't rate-limit guesses. The encryption is
  only as good as the password.
- `src/content.html` is git-ignored on purpose, so the plaintext plan stays off GitHub.
  **Keep your own backup of it** — it's the only editable source.
- Decryption needs a **secure context**: works on GitHub Pages (HTTPS) and `localhost`,
  but opening `index.html` straight from disk (`file://`) may fail. To test locally:
  ```bash
  npx serve .        # or:  python3 -m http.server
  ```
- No password recovery — if forgotten, just rebuild with a new password.
