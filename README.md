# South of France Trip — Travel Plan

A single-page travel itinerary for the Gourdon villa trip, ready to host on **GitHub Pages**.

## Files

| File | Purpose |
|------|---------|
| `index.html` | Published site. Generated by the build — commit this. |
| `src/content.html` | Main itinerary markup. |
| `src/stargazing.html`, `src/shopping.html` | Extra page fragments injected at build time. |
| `assets/content.css`, `assets/content.js` | Styles and interactivity. |
| `build.mjs` | Stitches source fragments into `index.html`. |
| `package.json` | Convenience `build` script. |
| `supabase/shopping_items.sql` | Supabase table + RLS policies for the shared shopping list. |

## Build

Requires **Node 22**.

```bash
node build.mjs
# or
npm run build
```

This regenerates `index.html` from `src/content.html` plus the stargazing and shopping fragments. Edit the source files, then rebuild before publishing.

## Shared shopping list

The **Shopping** page (`#shopping`) lets everyone add villa groceries and tick items off when bought. Purchased items are hidden by default; use **Show purchased** to review them. Items are auto-grouped by supermarket aisle.

To sync for the whole group via [Supabase](https://supabase.com/):

1. Create a Supabase project.
2. Run `supabase/shopping_items.sql` in the SQL editor (creates the table and row-level security policies).
3. Enable **Realtime** for `shopping_items` (or run `alter publication supabase_realtime add table public.shopping_items;`).
4. From **Project Settings → API**, copy the **Project URL** and **anon public** key.
5. Paste them into `data-supabase-url` and `data-supabase-anon-key` on the `#page-shopping` element in `src/shopping.html`.
6. Rebuild and redeploy.

The anon key is public client-side (normal for Supabase). RLS allows anyone with the key to read and update the list — fine for a shared trip list; tighten policies later if needed.

There is no cap on how many items can be added.

## Deploy to GitHub Pages

```bash
git add .
git commit -m "Update travel plan"
git push origin main
```

Then in the repo: **Settings → Pages → Source: Deploy from a branch**, branch **main**,
folder **/ (root)**. The site appears at `https://<you>.github.io/<repo>/` within a minute or two.

## Local preview

```bash
npx serve .
# or:  python3 -m http.server
```
